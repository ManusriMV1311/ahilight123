Frequently Asked Questions

Common questions about CyberFortress capabilities, deployment, and procurement.

General Questions

What is CyberFortress?

CyberFortress is an autonomous security platform that combines threat detection, automated response, deception, and immutable audit capabilities in a single unified platform.


What makes CyberFortress different from existing SIEM/SOAR tools?

Behavioral correlation across domains: We correlate network, endpoint, and identity data using graph neural networks to detect attack chains. Traditional SIEMs correlate within domains but miss cross-domain attacks.

Autonomous detection: Our ML models reduce false positives to <1% compared to 30-50% for rule-based SIEMs.

Policy-driven automation: Response actions execute automatically with human oversight, reducing MTTR by 85%.

Immutable audit: Blockchain-backed audit trails provide tamper-proof evidence for compliance and forensics.


Is this an EDR replacement?

No. CyberFortress complements EDR by correlating endpoint alerts with network and identity data. We integrate with leading EDR solutions (CrowdStrike, Carbon Black, SentinelOne) rather than replacing them.


Is this a SIEM replacement?

Partially. CyberFortress provides threat detection and correlation superior to traditional SIEMs, but we're not a general-purpose log management platform. Many customers use CyberFortress for security detection and keep their SIEM for compliance logging and long-term retention.





Technical Questions

What data does CyberFortress collect?

CyberFortress collects network flow data (NetFlow, sFlow, IPFIX), endpoint telemetry (via EDR integrations or eBPF agents), authentication logs (from IAM systems), and security tool alerts (from SIEM, firewall, etc.). We do NOT collect full packet captures or PII without explicit customer configuration.


What is the performance overhead?

eBPF-based edge agents have <1% CPU overhead. Network tap deployments have zero performance impact on production systems. Cloud-based processing eliminates on-premise infrastructure requirements for SaaS deployments.


How does behavioral baselining work?

CyberFortress observes normal activity for 7-14 days to establish baselines for user behavior, network patterns, and application interactions. ML models then detect deviations from these baselines. Baselines adapt over time as your environment evolves.


Can I customize detection rules?

Yes. While our ML models handle most detection, you can create custom rules using our Python-based plugin framework. You can also adjust sensitivity thresholds for existing detections.


How accurate is the detection?

Our false positive rate is <1% based on production deployments. This is 30-50x better than rule-based SIEMs. However, accuracy depends on environment-specific factors (network complexity, baseline quality, etc.).





Deployment Questions

How long does deployment take?

SaaS: Agent installation: <1 day. Integration setup: <1 week. Baselining: 1-2 weeks. Total time to production: 3-4 weeks.

On-premise: Infrastructure setup: 1-2 weeks. Agent installation: <1 day. Integration setup: 1 week. Baselining: 1-2 weeks. Total time to production: 4-6 weeks.


What are the infrastructure requirements?

For SaaS deployments, no on-premise infrastructure required (agents only). For on-premise deployments, see our deployment documentation for detailed specs. Minimum: 16 cores, 64 GB RAM, 1 TB SSD.


Do you support air-gapped networks?

Yes. On-premise deployment supports fully air-gapped environments with offline model updates delivered via secure file transfer.


Can I start with a POC?

Yes. We offer 30-day proof of concept deployments for qualified customers. POC includes full platform access, integration support, and dedicated SE resources.
