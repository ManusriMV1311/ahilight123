Deployment Options

Flexible deployment models to meet your security, compliance, and infrastructure requirements.

Deployment Models

SaaS (Recommended)

Fully managed cloud deployment with automatic scaling, updates, and 24/7 monitoring. No infrastructure management required. Fastest time to value.


Best For: Mid-to-large enterprises, technology companies

On-Premise

Complete control over infrastructure and data. Supports air-gapped networks and custom security requirements. Customer-managed updates and maintenance.


Best For: Government, critical infrastructure, regulated industries

Hybrid

Data collection and initial processing on-premise with cloud-based ML inference. Balances security requirements with cloud scalability benefits.


Best For: Enterprises with mixed security requirements




System Requirements

Minimum Requirements (On-Premise)

  • CPU: 16 cores (x86-64)
  • RAM: 64 GB
  • Storage: 1 TB SSD (RAID 10 recommended)
  • Network: 1 Gbps bandwidth per 1000 devices

Recommended Requirements (On-Premise)

  • CPU: 32 cores with GPU acceleration (NVIDIA A100 or similar)
  • RAM: 256 GB
  • Storage: 10 TB NVMe SSD (RAID 10)
  • Network: 10 Gbps for high-volume deployments




Installation Process

Deploy Agents

Install CyberFortress agents across infrastructure using Ansible playbooks or Docker containers. Typical deployment time: <1 hour for 1000 endpoints.

Configure Integrations

Connect existing security tools (SIEM, EDR, firewalls) via pre-built connectors or REST API. Test data flow and verify connectivity.

Baseline Normal Behavior

CyberFortress observes 7-14 days of normal activity to establish behavioral baselines. No action required during this period.

Enable Detection (Alerts Only)

Start with detection-only mode to validate accuracy and tune alert thresholds. Review and adjust based on false positive rate.

Enable Automated Response

Gradually enable automated response actions, starting with low-risk actions (e.g., logging, alerting) and progressively adding blocking/isolation as confidence increases.





Support Model

Onboarding

Dedicated Customer Success Manager (CSM) guides 30-day onboarding with weekly check-ins, configuration assistance, and training sessions.


Support SLA

  • Critical Issues: <1 hour response time (24/7/365)
  • High Priority: <4 hours response time
  • Normal: <24 hours response time

Training

  • Online documentation and video tutorials
  • Monthly webinars for new features
  • On-site training for enterprise customers (optional)
  • Certification program for security analysts




Rollback & Safety

Response Rollback

Automatic undo for failed response actions. If automated response causes system issues, CyberFortress can reverse actions (unblock IP, restore user access, etc.).


Safe Mode

Detection-only mode disables all automated responses while maintaining full visibility. Use during change windows or when troubleshooting.


Emergency Override

Security team can pause all automated actions with single button click. Human override always takes precedence over automation.
